# 授权接口

本文档提供了系统授权相关的API接口说明。

## 1. 获取RSA公钥

> 获取服务器端RSA公钥，用于客户端对敏感数据（如密码）进行加密

### 基本信息

- **接口URL**: `/auth/public-key`
- **请求方式**: `GET`
- **认证要求**: 否
- **权限要求**: 无

### 请求参数

无

### 响应数据

#### 成功响应

状态码: `200000`

```json
{
  "code": 200000,
  "message": "操作成功",
  "data": {
    "publicKey": "-----BEGIN PUBLIC KEY-----\nMIIBIjANBgkqhki...(公钥内容)...IDAQAB\n-----END PUBLIC KEY-----"
  },
  "path": "/auth/public-key",
  "timestamp": 1672574400
}
```

##### 响应字段说明

| 字段名    | 类型   | 描述                 |
| --------- | ------ | -------------------- |
| publicKey | string | RSA公钥（PEM格式）   |

### 示例

#### 请求示例

```bash
curl -X GET "https://api.example.com/auth/public-key"
```

#### 响应示例

```json
{
  "code": 200000,
  "message": "操作成功",
  "data": {
    "publicKey": "-----BEGIN PUBLIC KEY-----\nMIIBIjANBgkqhki...(公钥内容)...IDAQAB\n-----END PUBLIC KEY-----"
  },
  "path": "/auth/public-key",
  "timestamp": 1672574400
}
```

## 2. 用户登录

> 用户登录并获取访问令牌和刷新令牌

### 基本信息

- **接口URL**: `/auth/login`
- **请求方式**: `POST`
- **认证要求**: 否
- **权限要求**: 无

### 请求参数

#### 请求体 (JSON)

```json
{
  "username": "admin",
  "password": "加密后的密码字符串"
}
```

| 字段名   | 类型   | 是否必须 | 描述                                        | 示例值                |
| -------- | ------ | -------- | ------------------------------------------- | --------------------- |
| username | string | 是       | 用户名                                      | "admin"               |
| password | string | 是       | 经过RSA加密的密码（Base64编码字符串）      | "加密后的密码字符串"  |

### 响应数据

#### 成功响应

状态码: `200000`

```json
{
  "code": 200000,
  "message": "登录成功",
  "data": {
    "accessToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "refreshToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
  },
  "path": "/auth/login",
  "timestamp": 1672574400
}
```

##### 响应字段说明

| 字段名       | 类型   | 描述                                  |
| ------------ | ------ | ------------------------------------- |
| accessToken  | string | JWT访问令牌，用于访问需要认证的接口  |
| refreshToken | string | JWT刷新令牌，用于获取新的访问令牌    |

#### 错误响应

可能的错误状态码：`400000`, `422000`

```json
{
  "code": 400000,
  "message": "用户名或密码错误",
  "data": null,
  "path": "/auth/login",
  "timestamp": 1672574400
}
```

### 示例

#### 请求示例

```bash
curl -X POST \
  "https://api.example.com/auth/login" \
  -H "Content-Type: application/json" \
  -d '{
    "username": "admin",
    "password": "加密后的密码字符串"
  }'
```

#### 响应示例

```json
{
  "code": 200000,
  "message": "登录成功",
  "data": {
    "accessToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "refreshToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
  },
  "path": "/auth/login",
  "timestamp": 1672574400
}
```

### 注意事项

- 请求中的密码字段必须使用从`/auth/public-key`接口获取的RSA公钥进行加密
- 加密后的内容需要转换为Base64格式的字符串

## 3. 刷新令牌

> 当访问令牌过期时，使用刷新令牌获取新的令牌对

### 基本信息

- **接口URL**: `/auth/refresh`
- **请求方式**: `POST`
- **认证要求**: 否
- **权限要求**: 无

### 请求参数

#### 请求体 (JSON)

```json
{
  "refreshToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
}
```

| 字段名       | 类型   | 是否必须 | 描述               | 示例值                             |
| ------------ | ------ | -------- | ------------------ | ---------------------------------- |
| refreshToken | string | 是       | JWT刷新令牌        | "eyJhbGciOiJIUzI1NiIsInR5cCI6..." |

### 响应数据

#### 成功响应

状态码: `200000`

```json
{
  "code": 200000,
  "message": "令牌刷新成功",
  "data": {
    "accessToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "refreshToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
  },
  "path": "/auth/refresh",
  "timestamp": 1672574400
}
```

##### 响应字段说明

| 字段名       | 类型   | 描述                               |
| ------------ | ------ | ---------------------------------- |
| accessToken  | string | 新的JWT访问令牌                    |
| refreshToken | string | 新的JWT刷新令牌                    |

#### 错误响应

可能的错误状态码：`401003`, `401004`, `422000`

```json
{
  "code": 401003,
  "message": "刷新令牌已过期",
  "data": null,
  "path": "/auth/refresh",
  "timestamp": 1672574400
}
```

### 示例

#### 请求示例

```bash
curl -X POST \
  "https://api.example.com/auth/refresh" \
  -H "Content-Type: application/json" \
  -d '{
    "refreshToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
  }'
```

#### 响应示例

```json
{
  "code": 200000,
  "message": "令牌刷新成功",
  "data": {
    "accessToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "refreshToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
  },
  "path": "/auth/refresh",
  "timestamp": 1672574400
}
```

## 4. 获取当前用户信息

> 获取当前登录用户的详细信息

### 基本信息

- **接口URL**: `/auth/info`
- **请求方式**: `GET`
- **认证要求**: 是
  - 使用 `JwtAuthGuard` 进行JWT认证
- **权限要求**: 无

### 请求参数

无

### 响应数据

#### 成功响应

状态码: `200000`

```json
{
  "code": 200000,
  "message": "操作成功",
  "data": {
    "id": "1",
    "username": "admin"
  },
  "path": "/auth/info",
  "timestamp": 1672574400
}
```

##### 响应字段说明

| 字段名   | 类型   | 描述                 |
| -------- | ------ | -------------------- |
| id       | string | 用户ID               |
| username | string | 用户名               |

#### 错误响应

可能的错误状态码：`401000`, `401001`, `401002`

```json
{
  "code": 401001,
  "message": "访问令牌已过期",
  "data": null,
  "path": "/auth/info",
  "timestamp": 1672574400
}
```

### 示例

#### 请求示例

```bash
curl -X GET \
  "https://api.example.com/auth/info" \
  -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
```

#### 响应示例

```json
{
  "code": 200000,
  "message": "操作成功",
  "data": {
    "id": "1",
    "username": "admin"
  },
  "path": "/auth/info",
  "timestamp": 1672574400
}
```

## 5. 用户登出

> 用户登出系统，使当前令牌失效

### 基本信息

- **接口URL**: `/auth/logout`
- **请求方式**: `POST`
- **认证要求**: 是
  - 使用 `JwtAuthGuard` 进行JWT认证
- **权限要求**: 无

### 请求参数

无

### 响应数据

#### 成功响应

状态码: `200000`

```json
{
  "code": 200000,
  "message": "登出成功",
  "data": null,
  "path": "/auth/logout",
  "timestamp": 1672574400
}
```

#### 错误响应

可能的错误状态码：`401000`, `401001`, `401002`

```json
{
  "code": 401001,
  "message": "访问令牌已过期",
  "data": null,
  "path": "/auth/logout",
  "timestamp": 1672574400
}
```

### 示例

#### 请求示例

```bash
curl -X POST \
  "https://api.example.com/auth/logout" \
  -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
```

#### 响应示例

```json
{
  "code": 200000,
  "message": "登出成功",
  "data": null,
  "path": "/auth/logout",
  "timestamp": 1672574400
}
```

### 注意事项

- 登出操作会使当前用户的访问令牌和刷新令牌全部失效
- 登出后需要重新登录才能获取新的令牌
